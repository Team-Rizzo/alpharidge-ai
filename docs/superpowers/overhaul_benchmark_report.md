# Article-Intelligence Quality Overhaul — Benchmark Report

**Date:** 2026-06-17 · **Branch:** `article-intelligence-v2` (alpharidge-ai) + `feat/call2b-keywords-sentiment`/`feat/...` (eval)
**Status:** Implemented, tested, benchmarked. **Not committed to main / not pushed** — for your review.

## What was fixed (the four reported defects)

| # | Defect | Fix (all deterministic, validator-safe) |
|---|--------|------|
| P1 | `narrative_keywords` hallucinated (every article got generic crypto/AI slugs) | Removed from LLM Call 2. Slugs now selected from the taxonomy by embedding similarity (all-MiniLM-L6-v2) with abstention, **plus a crypto-presence gate** so crypto-only narratives can't fire on non-crypto articles. |
| P2 | Uppercase-acronym ticker FPs (LEO, ALL, CAR, GOLD…) | Case-sensitive tickers that are common words or blocklisted acronyms are now ambiguous AND non-corroborating, so they need a cashtag / distinctive name / other evidence to survive. |
| P3 | Per-asset `direction` always neutral | Match an asset's sentences by the surface forms that appear in prose ("Tesla"), not the ticker symbol ("TSLA"); fall back to article-level sentiment instead of hard neutral. |
| P4 | Entity noise (Medtronic ×4; "Anthropic principle", "Elon Musk-backed") | Dedupe entities by canonical name+type (merging role/sentiment); trim adjectival/appositive NER fragments back to the entity. |

## Benchmarks

### Validation environment caveat
This box has **GLiNER / Flair / ReFinED uninstalled** (only spaCy + FinBERT active), so absolute recall is lower than production. All before/after numbers were taken on this same config, so **deltas are valid**; absolute values are not production-representative.

### Hand-curated targeted set (25 cases) — the ground-truth judge
`tests/fixtures/handcurated_overhaul.jsonl`, run by `tests/test_overhaul_regressions.py::test_handcurated_targeted`.
**Result: all 25 cases pass, zero soft-misses.** Confirms, on real prose:
- LEO (Low Earth Orbit), ALL, CAR, GOLD, IT excluded as tickers; `$LEO` and "Barrick Gold" (GOLD) still detected.
- TSLA/AAPL clear-bullish → bullish, clear-bearish → bearish (per-asset direction now carries signal).
- Medtronic ×4 → 1; "Anthropic principle" → "Anthropic"; "Elon Musk-backed" → "Elon Musk".
- No crypto narrative slugs on the VC-interview / dividend / SpaceX / AI-shopping articles.

### Gold-set metrics (eval, GLM-5.1, before → after)

| Field | Metric | Before | After | Read |
|-------|--------|--------|-------|------|
| `narrative_keywords` | Jaccard vs GLM | ~0.000 (YAKE/KeyBERT) | **0.374** | ~71% of the GLM-vs-Opus ceiling (0.527), with zero LLM. Crypto gate nudged 0.371→0.374. |
| `entities` | F1 (name.lower) | 0.4548 | **0.5755** | **+26%** — real win from dedup + fragment trim. |
| `assets` | F1 (ticker) | 0.1988 | 0.1891 | Slightly down — see note. |
| `assets` | direction agreement | 0.4005 | 0.3661 | Down — see note. |

**Why the `assets` gold numbers are not informative (and the curated set is):**
The gold labels were produced by the *same analyzer with the old, buggy code*. So gold's `assets` list itself contains the LEO/GOLD false positives we now remove (our correct output diverges → F1 dips), and gold's per-asset `direction` is itself almost entirely "neutral" (it was generated by the always-neutral bug), so our newly-real directions "disagree" with buggy gold (agreement dips). This is exactly the blind-spot we anticipated when choosing "existing gold + hand-curated set" — for P2/P3 the curated set is the valid judge, and it passes.

## Calibration
- Narrative threshold `tau = 0.30` (config-overridable via `NARRATIVE_TAU`), chosen by sweep maximizing mean Jaccard. Predictor abstains more than gold (74% vs 32% empty) — the *safe* failure mode given the original bug was over-emitting.

## Known limitations / follow-ups
- Embedding narrative selection still can't reach the 30 gold-only slugs absent from the 38-slug taxonomy (≈1% of gold instances) and under-covers vs gold. A hybrid LLM-verify or richer centroids could lift recall if desired (deferred — would add LLM scope against the kill-Call-2 direction).
- Install GLiNER/Flair/ReFinED and re-run for production-representative absolutes.
- Minor (from task-1 review, deferred): `overhaul_bench.py` lacks a guard for an unknown `--fields` value.
- Per-asset direction on genuinely *mixed* articles leans to FinBERT's majority vote (can read bearish where a human says neutral) — acceptable; the bug (stuck-neutral) is fixed.
